# Move `@vaneui/mcp` to a standalone repo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Relocate the `@vaneui/mcp` package from `C:/GitHub/vaneui/mcp/` (subfolder of the component library repo) to the standalone clone at `C:/GitHub/vaneui-mcp/` (already git-cloned from `https://github.com/vaneui/mcp.git`, currently empty except for a placeholder README). After the move, the MCP server keeps syncing docs from `vaneui/` and `vaneui-web/` via sibling-folder lookup.

**Architecture:** The MCP package is fully self-contained (3 TS source files, 2 Node sync scripts, a `package.json`, a `tsconfig.json`, README, and generated `resources/` + `dist/` that are gitignored). The only cross-repo coupling is the two sync scripts that read `vaneui/CLAUDE.md` / `vaneui/.claude/rules/*.md` / `vaneui-web/app/docs/data/**.{md,tsx}`. Today they compute those paths relative to their own location: `REPO_ROOT = ../` (vaneui), `vaneui-web = ../../vaneui-web`. After the move, both become `../vaneui` and `../vaneui-web` — sibling folders of `vaneui-mcp`. Two line edits per script. Docs and CLAUDE.md references get swept separately.

**Tech Stack:** Node.js ≥ 20, TypeScript 5.8, `@modelcontextprotocol/sdk`, ESM modules, no test framework (the package is currently untested — out of scope for this migration).

**Preservation of git history:** NOT attempted. The target repo (`vaneui-mcp`) is a fresh clone from an empty remote; the user explicitly said "newly cloned". We commit the migrated files as a single "Initial port from vaneui/mcp" commit. If history preservation becomes important later, a follow-up `git subtree split` PR is possible — but this plan does not attempt it.
